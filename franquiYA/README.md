# FranquiYA

Internal operations dashboard prototype for a franchise-style business.

## Overview

FranquiYA is a full-stack portfolio project focused on practical store operations: inventory, invoice processing, staff management, shifts, audits, tasks and dashboard views.

The project uses realistic business workflows, but the public repository should be treated as a demo/prototype. Local data, uploaded documents and generated databases are intentionally ignored from version control.

## Features

- Operational dashboard with KPIs
- Stock views and low-stock alerts
- Invoice upload and processing flow
- Mobile-first inventory audit interface
- Employee, role and shift management
- Holiday and task tracking
- Authentication with role-based access
- Optional AI-assisted invoice/chat flows through external APIs

## Tech stack

- **Frontend**: Next.js 14, React, TypeScript, Tailwind CSS
- **Backend**: FastAPI, SQLAlchemy, Pydantic
- **Database**: SQLite for local development
- **Integrations**: OpenWeatherMap API, Groq API, pdfplumber
- **Testing**: Jest, Testing Library, pytest

## Quick start

### Backend

```bash
cd franquiYA/backend
python -m venv venv
venv\Scripts\pip install -r requirements.txt
venv\Scripts\uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

### Frontend

```bash
cd franquiYA/frontend
npm install
npm run dev
```

## Demo users

## Environment variables

Create a local `backend/.env` file if you want to enable optional integrations:

```bash
DATABASE_URL=sqlite:///./franquiya.db
JWT_SECRET=change-this-local-secret
OPENWEATHERMAP_API_KEY=
GROQ_API_KEY=
CORS_ORIGINS=http://localhost:3000
```

## Repository hygiene

Generated databases, local backups, uploaded PDFs, local prompts, logs and environment files are not required to run the portfolio version and should stay out of Git.
